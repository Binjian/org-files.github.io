2.3. submodule

2.3.1. just update, track the new changes of the submodules (no change)

  1. manually : git fetch
  2. automatically: git submodule update –remote
    1. set specific branch .gitmodules submodule.submudlename.branch stable
    2. -f: tracked; no -f locally, not tracked.
    3. git config status.submodulesummary 1

2.3.2. work on the submodule collaboratively

  1. git submodule update –remote –rebase/–merge
  2. caveats: has to commit locally.
  3. publishing submodule changes: git push –recurse-submodules=check/on-demand

2.3.3. caveats:

  1. accidental revert: git submodule update –remote to update .gitmodule; and commit in parent projects.
  2. cannot merge: no submodule merge
